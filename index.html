Security Levels (SL 0 - SL 4)

SL 0: No Protection None

No specific requirements or security protection.

Use Case: A completely isolated demo unit with no network connection.

SL 1: Casual / Coincidental Mistake

Protection against casual or coincidental violation.

Use Case: An employee accidentally connecting a wrong cable or a generic mass-market virus.

SL 2: Simple Intentional Script Kiddie

Protection against intentional violation using simple means/resources.

Use Case: A disgruntled insider with low skills or an attacker using standard downloadable exploit tools (e.g. Metasploit).

SL 3: Sophisticated Intentional Professional

Protection against intentional violation using sophisticated means and moderate resources.

Use Case: A targeted attack by a hacktivist group or professional cybercrime gang (e.g., targeted Ransomware).

SL 4: Nation State APT

Protection against intentional violation using sophisticated means and extended resources.

Use Case: A state-sponsored campaign (e.g., Stuxnet, Industroyer) targeting critical national infrastructure.

compare_arrows Why Transition to IEC 62443?

Purdue Model Structure

Focuses on network hierarchy. Assumes trust within layers. Often rigid against modern IIoT/Cloud flows.

IEC 62443 Risk

Focuses on security zones based on criticality. Supports Zero Trust (Conduits). Flexible for complex, lateral data flows.

Key Benefit: IEC 62443 allows you to place a critical controller in a high-security zone regardless of its "Level", protecting against lateral movement that Purdue misses.

BASIC

Assets tracked via static spreadsheets or manual walkdowns; high risk of obsolescence and 'shadow OT'.
Use Case: A technician manually updates an Excel sheet after replacing a faulty PLC, often forgetting to record firmware versions.

INTERMEDIATE

Periodic active scanning performed during maintenance windows; inventory is digitized but often outdated between scans.
Use Case: Running a Nessus scan during the annual shutdown to capture asset details for compliance audits.

ADVANCED

Real-time visibility using continuous passive network monitoring (DPI) to identify assets and communication flows without disrupting operations.
Use Case: Claroty or Nozomi passively identifying a new HMI connected to the network instantly.

PROFESSIONAL

Automated hybrid discovery combining passive monitoring, safe active queries, and API integration with CMDB/ERP for a unified, real-time asset source of truth.
Use Case: ServiceNow automatically triggering a ticket when a new device is detected, enriching it with CVE data from the OT IDS.

BASIC

Flat network architecture with bridged IT/OT environments, allowing unrestricted lateral movement and high blast radius.
Use Case: Ransomware from a reception PC spreads directly to the SCADA server because they share the same subnet.

INTERMEDIATE

Perimeter defense implemented via an Industrial DMZ (IDMZ) with firewalls separating Enterprise (L4) from Operations (L3).
Use Case: Preventing direct RDP access from the Corporate LAN to the Plant Floor; forcing traffic through a jump host.

ADVANCED

Internal segmentation enforcing the IEC 62443 Zones & Conduits model to isolate critical processes and restrict east-west traffic.
Use Case: Placing the Safety Instrumented System (SIS) in a dedicated zone that blocks all traffic except from the Engineering Station.

PROFESSIONAL

Micro-segmentation and Identity-Based Zero Trust Network Access (ZTNA) policies applied down to individual workloads and endpoints.
Use Case: An Operator's iPad can only access the specific HMI screen for Line 1, authenticated via certificate and context, blocking all other IPs.

BASIC

Widespread use of shared vendor accounts and default passwords; no individual accountability for actions.
Use Case: All operators login to the HMI using the 'admin/1234' credentials taped to the monitor.

INTERMEDIATE

Unique user accounts implemented with Role-Based Access Control (RBAC) managed locally on devices.
Use Case: Creating individual local accounts for 'Operator', 'Engineer', and 'Maintenance' on a specific HMI panel.

ADVANCED

Centralized authentication (AD/LDAP) integrated with OT systems; Multi-Factor Authentication (MFA) required for remote access.
Use Case: Engineers use their corporate Active Directory credentials to log in to the Historian, enforced by Group Policy.

PROFESSIONAL

Privileged Access Management (PAM) with Just-in-Time (JIT) provisioning, session recording, and behavioral monitoring.
Use Case: A vendor requests temporary admin access to a PLC; CyberArk grants access for 2 hours and video-records the entire session.

BASIC

Uncontrolled use of consumer-grade tools (TeamViewer, AnyDesk) allowing direct, unmonitored connections to control systems.
Use Case: A vendor installs TeamViewer on a Critical Engineering Workstation to fix an issue from home without IT knowledge.

INTERMEDIATE

VPN connections terminating at a Jump Host in the IDMZ; basic logging of connections but limited session visibility.
Use Case: Maintenance staff VPN into a DMZ server, then RDP into the plant floor; logs show *who* connected but not *what* they did.

ADVANCED

Secure Remote Access (SRA) proxy enforcement requiring MFA, protocol isolation, and approval workflows.
Use Case: Claroty SRA requires a plant manager to approve a vendor's request before the connection is established.

PROFESSIONAL

Zero Trust Network Access (ZTNA) with ephemeral connectivity, full session video recording, and real-time termination capabilities.
Use Case: Zscaler Private Access allows a specific application tunnel for a vendor to update a PLC, blocking network scanning, with full forensic recording.

BASIC

Reactive approach; patching performed only ad-hoc or in response to major public incidents.
Use Case: Scrambling to patch BlueKeep only after hearing about it on the news, with no prior inventory of vulnerable Windows 7 machines.

INTERMEDIATE

Annual penetration testing and periodic vulnerability assessments; remediation is often delayed until scheduled shutdowns.
Use Case: Hiring a third-party pen-tester once a year to find open ports and outdated firmware.

ADVANCED

Continuous vulnerability scoring based on asset context and passive monitoring; risk-based prioritization of remediations.
Use Case: Tenable.ot identifies a CVE in a PLC but marks it 'Low Priority' because the device is behind a firewall and locked key-switch.

PROFESSIONAL

Threat-Informed Defense using SSVC (Stakeholder-Specific Vulnerability Categorization) and automated mitigation strategies.
Use Case: Automatically applying an IPS signature to block traffic exploiting a new Zero-Day before the vendor patch is even released.

BASIC

No formal version control; changes made locally on devices without backups or documentation.
Use Case: An engineer changes a setpoint on a PLC to fix a glitch but doesn't save the project file; the change is lost after a power cycle.

INTERMEDIATE

Manual backups stored centrally; periodic manual checks for configuration deviations.
Use Case: Monthly routine where technicians plug into PLCs, upload the logic, and save it to a shared network drive with a date stamp.

ADVANCED

Automated change detection systems that alert on unauthorized modifications to controller logic or firmware.
Use Case: Asset Centre or MDT AutoSave triggers an email alert to the CISO when a PLC's logic block is modified at 2 AM.

PROFESSIONAL

Real-time integrity monitoring with automated rollback capabilities and strict enforcement of 'Golden Image' baselines.
Use Case: Tripwire instantly detecting a change in a critical HMI config file and automatically reverting it to the known-good state.

BASIC

Ad-hoc response relying on individual heroics; no formal IR plan or documentation exists.
Use Case: Calling the control room supervisor on their personal cell phone when screens go black, hoping they know what to do.

INTERMEDIATE

Defined IR procedures and call trees; basic log retention for retrospective analysis.
Use Case: Following a printed binder checklist to disconnect the network cable when a virus is suspected.

ADVANCED

OT-specific playbooks developed for common scenarios (e.g., ransomware, controller manipulation); regular tabletop exercises.
Use Case: Conducting a 'Ransomware in the Control Room' simulation with IT, OT, and Legal teams to practice decision making.

PROFESSIONAL

Integrated Security Orchestration, Automation and Response (SOAR) with a unified IT/OT SOC and automated containment workflows.
Use Case: Cortex XSOAR automatically isolating a compromised HMI from the network via switch API calls upon detecting malicious C2 traffic.

BASIC

Reliance on basic firewall logs and antivirus alerts; no visibility into process-level anomalies.
Use Case: Only knowing about an infection when the McAfee icon turns red on the engineering workstation.

INTERMEDIATE

Centralized logging (Syslog) to an IT SIEM; basic correlation rules for known IT-centric threats.
Use Case: Forwarding Windows Event Logs from HMIs to Splunk to catch multiple failed login attempts.

ADVANCED

Deployment of OT-aware IDS with Deep Packet Inspection (DPI) to detect industrial protocol anomalies and operational violations.
Use Case: Alerting when a 'CIP Stop CPU' command is sent to a PLC from an unauthorized IP address.

PROFESSIONAL

Advanced behavioral analytics (UEBA) and AI-driven threat hunting to identify sophisticated 'living off the land' attacks.
Use Case: AI detecting that a specific workstation is reading 500% more PLC tags than usual, indicating data exfiltration.

BASIC

Systems remain unpatched indefinitely to avoid downtime; updates applied only upon hardware failure.
Use Case: Running Windows XP on mission-critical HMIs because 'it works' and patching requires a reboot.

INTERMEDIATE

Regular patching cadence established during annual maintenance shutdowns; reliance on vendor-approved lists.
Use Case: Waiting for the July shutdown to apply Microsoft patches that were approved by Siemens in January.

ADVANCED

Risk-based patching prioritization; implementation of compensatory controls for legacy systems that cannot be patched.
Use Case: Locking down firewall rules for a legacy controller instead of patching it, documenting this as an accepted risk.

PROFESSIONAL

Virtual patching implemented at the network level (IPS) and automated testing pipelines to validate updates before deployment.
Use Case: Using a Palo Alto Networks firewall to block the exploit path for a new Windows vulnerability, protecting unpatched HMIs immediately.

BASIC

Implicit trust in all vendors and integrators; no security vetting of third-party hardware or software.
Use Case: Plugging in a contractor's USB drive containing 'updated software' without scanning it.

INTERMEDIATE

Vendor security questionnaires utilized during procurement; basic remote access stipulations in contracts.
Use Case: Asking the PLC vendor to fill out a SIG Lite questionnaire before signing the support contract.

ADVANCED

Enforced security requirements in SLAs; collection of Software Bill of Materials (SBOM) for critical assets.
Use Case: Rejecting a new RTU delivery because the vendor failed to provide an SBOM listing all open-source libraries used.

PROFESSIONAL

Continuous third-party risk monitoring; automated SBOM analysis for zero-day risks; strict validation of vendor remote support activities.
Use Case: Automatically flagging a vendor component as 'High Risk' in the asset inventory when a Log4j vulnerability is discovered in its SBOM.

BASIC

Unrestricted use of USB drives and portable media; high risk of malware introduction via air-gap bridging.
Use Case: Operators charging their personal phones on the HMI USB ports or bringing movies to watch on night shift.

INTERMEDIATE

Written policies prohibiting unauthorized USB use; physical port blockers applied to critical HMIs.
Use Case: Gluing USB ports shut or using plastic locks to prevent insertion of drives.

ADVANCED

Deployment of decontamination kiosks ('Sheepdip' stations) to scan and sanitize media before use.
Use Case: A contractor must scan their USB drive at a kiosk, which copies clean files to a secure corporate USB, before entering the plant.

PROFESSIONAL

Technical enforcement via endpoint controls (allow-listing) and hardware-based unidirectional data transfer appliances.
Use Case: Windows Group Policy blocking all USB storage devices except those with specific Hardware IDs encrypted by the company.

BASIC

IT security policies loosely applied to OT without adaptation; generic security awareness training.
Use Case: Forcing operators to change complex passwords every 30 days, leading to passwords written on post-it notes.

INTERMEDIATE

Specific OT security policies defined; role-based training provided for engineering and operations staff.
Use Case: Conducting specific training for engineers on 'How to securely transfer files to the secure zone'.

ADVANCED

Dedicated OT security leadership (CISO/Security Manager); regular internal audits and compliance tracking.
Use Case: Appointing an 'OT Security Officer' for each plant who reports to both Plant Management and the Global CISO.

PROFESSIONAL

Board-level visibility of OT risk; cyber risk quantified in business terms; culture of continuous improvement and 'security by design'.
Use Case: The Board reviewing OT Cyber Risk alongside Financial Risk quarterly, approving budget for resilience based on loss expectancy.

BASIC

Reliance on physical keys and locks; unsupervised access to control rooms and cabinets.
Use Case: The server room key is kept under the doormat or in an unlocked drawer in the control room.

INTERMEDIATE

Electronic badge access systems implemented for critical areas; logs reviewed retrospectively.
Use Case: Requiring an RFID badge to enter the rack room, with logs checked only after a theft occurs.

ADVANCED

Integration of physical and logical access logs; CCTV monitoring correlated with system events.
Use Case: Correlating a badge swipe at the server room door with a local login event on the server console to detect tailgating.

PROFESSIONAL

Biometric authentication, anti-tailgating systems, and geofencing integrated with real-time security operations.
Use Case: Alerting the SOC if a user logs in to a SCADA terminal while their physical badge location shows them as 'Off Site'.

BASIC

No formal disaster recovery plan; reliance on manual restoration from ad-hoc backups.
Use Case: Trying to find the original installation CDs and license keys after a server hard drive failure.

INTERMEDIATE

Offsite backups maintained; basic recovery time objectives (RTO) defined but rarely tested.
Use Case: Storing backup tapes in a fireproof safe offsite and attempting a restore only during annual audits.

ADVANCED

Redundant hot-standby sites and systems; regular full-scale recovery drills simulating cyber-physical attacks.
Use Case: Failing over operations to a secondary data center during a planned drill to prove RTO of under 4 hours.

PROFESSIONAL

Resilience engineering principles applied; automated failover capabilities and 'Chaos Engineering' stress testing.
Use Case: Randomly disabling a critical historian service in production to verify self-healing scripts work as designed.

BASIC

Use of consumer-grade Wi-Fi with weak encryption (WEP/WPA) or open networks.
Use Case: Using a Best Buy router named 'Plant_Guest' connected to the OT network for easy iPad access.

INTERMEDIATE

WPA2 Personal encryption; SSID hiding and MAC address filtering implemented.
Use Case: Configuring a pre-shared key for handheld scanners and filtering access by their specific MAC addresses.

ADVANCED

WPA3 Enterprise (802.1x) authentication; strict segmentation of wireless traffic into isolated VLANs.
Use Case: Authenticating AGVs (Automated Guided Vehicles) using individual certificates, placing them in a restricted VLAN.

PROFESSIONAL

Continuous Wireless Intrusion Prevention Systems (WIPS) and spectrum analysis to detect and block rogue devices instantly.
Use Case: Automatically jamming the signal of a rogue Access Point ('Pineapple') detected near the perimeter fence.

BASIC

Ad-hoc risk decisions based on 'gut feeling' or driven solely by compliance audit findings.
Use Case: Installing an antivirus because the auditor said so, without considering if the machine can handle the CPU load.

INTERMEDIATE

Qualitative risk assessments (High/Medium/Low) performed annually using static spreadsheets.
Use Case: Rating 'Loss of View' as 'High Risk' in an Excel sheet during the annual risk review meeting.

ADVANCED

Semi-quantitative frameworks utilized; regular updates to the risk register based on threat landscape changes.
Use Case: Using the MITRE ATT&CK for ICS matrix to map current controls against known threat actor TTPs.

PROFESSIONAL

Quantitative Risk Assessment (e.g., FAIR model) used to calculate financial exposure; real-time dynamic risk scoring.
Use Case: Calculating that 'Patching this vulnerability costs $50k but reduces Annualized Loss Expectancy by $2M'.
